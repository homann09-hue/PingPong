# ADR 0005: Max-win, stake, and RNG disclosure boundary

## Status

Accepted.

## Decision

- Every published slot defines semantic math version, allowed integer stake
  steps, maximum win multiplier, and ordered win-class thresholds.
- The engine rejects unconfigured stakes even if an API caller bypasses the UI.
- Max-win is applied once inside deterministic settlement. Rounds after the cap
  are excluded, the payable final round is capped, and a typed event is emitted.
- Production entropy is generated by the API and persisted for controlled audit
  and replay. The seed is removed from public API responses.
- PostgreSQL records balance before/after, server version, math model version,
  complete result, and normalized spin events in the wallet transaction.

## Consequences

Clients cannot choose hidden stake values, infer future results from disclosed
seeds, or credit uncapped outcomes. Changing any probability, payout, stake, or
max-win behavior requires a new immutable math model version and fresh evidence.
